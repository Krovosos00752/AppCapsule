# AppCapsule

Изолированное пространство для запуска приложений на Android без root: приложения
внутри капсулы получают отдельный Android user id, отдельное хранилище и не
видят данные/файлы личного профиля. Это тот же системный механизм (Android
Work Profile), на котором построены Samsung Knox, Island и Shelter — только
не как отдельная OEM-прошивка, а как обычное приложение.

## Как это работает

1. **Создание пространства.** AppCapsule регистрирует себя как *Device Policy
   Controller* и через системный визард `ACTION_PROVISION_MANAGED_PROFILE`
   создаёт управляемый профиль (`android.app.admin.DevicePolicyManager`).
   AppCapsule становится *profile owner* этого профиля — не Device Owner, то
   есть у него нет контроля над остальным телефоном и личным профилем.
2. **Изоляция.** Всё, что происходит внутри профиля (файлы, буфер обмена,
   уведомления, установленные приложения), физически отделено ядром Linux —
   отдельный UID, отдельная файловая система. Это гарантия ОС, а не самого
   приложения.
3. **Наполнение пространства.** Копия AppCapsule, которая автоматически
   устанавливается внутрь нового профиля (т.к. является его profile owner),
   поднимает `VaultBridgeService`. Личная сторона (`MainActivity`) обращается
   к нему через `DevicePolicyManager#bindDeviceAdminServiceAsUser` — этот API
   существует специально для того, чтобы DPC мог общаться сам с собой поперёк
   границы профилей — и просит вызвать
   `DevicePolicyManager#installExistingPackage`, чтобы протащить уже
   установленное на телефоне приложение внутрь профиля без повторной загрузки.
4. **Запуск.** Системный лаунчер сам показывает для приложений, установленных
   в рабочем профиле, отдельный значок с бейджем в списке приложений — именно
   оттуда происходит запуск уже полностью изолированной копии. AppCapsule
   намеренно не пытается запускать чужие приложения кросс-профильно
   самостоятельно: `CrossProfileApps.startMainActivity` разрешён только для
   собственного пакета вызывающего приложения, если только это не
   приложение-лаунчер по умолчанию.

## Честные ограничения

- **Один вызов — один профиль.** Обычное Android-устройство поддерживает
  только один Work Profile на стороннее приложение без root/MDM. Это значит
  максимум 2 одновременных копии одного приложения (личная + 1 изолированная),
  не N. Настоящий мультиинстанс (3+ копий) потребовал бы root или
  OEM-привилегий — сознательно не реализовано.
- **VPN и подобное системное состояние по умолчанию видны профилю.** Сетевой
  стек общий для личного и рабочего профиля, если не настраивать per-profile
  VPN отдельно. В этой сборке не реализовано — по вашему решению это не
  приоритет.
- **Активация требует явного шага пользователя** (кнопка «Создать
  изолированное пространство» → системный визард), это не Play Store
  one-tap-installflow — так и должно быть, это официальный некорневой путь.
- Не проверялось на реальном устройстве/эмуляторе — в этом окружении нет
  Android SDK и сети до `dl.google.com`, поэтому сборка и рантайм-проверка
  доступны только у вас локально (см. ниже).

## Сборка

Нужен Android Studio (Koala/Ladybug или новее) с установленным Android SDK
(compileSdk/targetSdk 35, JDK 17).

1. Откройте папку проекта в Android Studio — она сама сгенерирует `gradlew`
   и синхронизирует зависимости (нужен доступ в интернет к `dl.google.com` и
   `repo.maven.apache.org`, в этом окружении он заблокирован, поэтому Gradle
   wrapper здесь сгенерировать не удалось).
2. Соберите и установите на реальное Android-устройство (эмулятор Play Store
   не всегда поддерживает управляемые профили) — `Run ▶`.
3. На телефоне откройте AppCapsule → «Создать изолированное пространство» →
   пройдите системный визард создания рабочего профиля.
4. После создания на личной стороне AppCapsule покажет список установленных
   приложений с кнопкой «Изолировать» — она клонирует выбранное приложение
   внутрь пространства.
5. Откройте приложение через бейджед-значок в общем списке приложений
   телефона (обычно отдельная вкладка «Рабочие» / иконка с точкой) — это и
   есть его изолированный экземпляр.

## Структура

```
app/src/main/java/com/appcapsule/vault/
  admin/          DeviceAdminReceiver (регистрация как profile owner)
  provisioning/   Обработчики системного визарда создания профиля
  cross/          Управление профилем + мост IPC между личной и рабочей стороной
  data/           Список установленных приложений личного профиля
  ui/             Compose UI
```
